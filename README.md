Trabalho realizado por Luis augusto Mortari e Richard Leonardo Otto


Perguntas para serem respondidas:
1. Qual a complexidade das operações em:
● Melhor caso
● Pior caso
2. Por que usar duas árvores pode ser melhor que uma só?
3. Quais problemas podem ocorrer se as árvores ficarem desbalanceadas?

   Resposta 1: O melhor caso ao meu ver é O(log n) que seria quando a arvore estivesse balanceada
               O pior caso teria uma complexidade O(n) que seria quando a arvore estivesse desbalanceada.

   Resposta 2: porque eu crio uma arvore para o ID e outra para o preço se so existesse uma arvore ficaria muito lento.

   Resposta 3: Uma arvore se ficar desbalanceada o problema é que ao buscar algum elemento ela fique lerda e a complexidade aumente.
